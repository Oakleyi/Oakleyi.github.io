    1. 1.1. 基础知识以及 Linux 入门
      1. 1.1.1. 第一阶段：2~3个月
        1. 1.1.1.1. 扎实算法和数据结构，刷LeetCode，刷出最优解
        2. 1.1.1.2. 扎实C语言：
        3. 1.1.1.3. 基本数据结构，
        4. 1.1.1.4. 指针：指针、指针步长、为啥能够通过指针找到对应数据、函数指针
      2. 1.1.2. 第二阶段：2~3个月
        1. 1.1.2.1. OO思想，用函数指针去搞定OO思想
        2. 1.1.2.2. Linux入门，几个常用命令，调试手段等
        3. 1.1.2.3. Linux c编程入门，多线程、多进程、内存管理、网络编程等等，Unix环境高级编程
      3. 1.1.3. 第三阶段：2~3个月
        1. 1.1.3.1. 学会封装思维，接口思维，多态思维等等
        2. 1.1.3.2. C++入门，掌握C with class，C++的OOP,两者的区别以及联系
        3. 1.1.3.3. 使用Linux C++将第二阶段的项目重写一遍
    2. 1.2. 第二大阶段：造轮子的能力
      1. 1.2.1. 第一阶段：2~3个月
        1. 1.2.1.1. 基础版高并发服务器，参考libevent、muduo等
        2. 1.2.1.2. 多线程、多进程、以及IO多路复用
        3. 1.2.1.3. 线程池、内存池的实现，为服务器增加组件
        4. 1.2.1.4. 日志组件
        5. 1.2.1.5. 消息队列–做限流削峰
      2. 1.2.2. 第二阶段：系统学习操作系统与分布式系统
        1. 1.2.2.1. 1、编写小型操作系统：参考清华大学操作系统课程的ucore 扎扎实实2~3个月
          1. 1.2.2.1.1. a)深入理解计算机系统（链接、装载那必须好好看）
        2. 1.2.2.2. 2、编写分布式系统：参考资料MIT的6.824课程，刷raft的论文，扎实搞2~3个月
    3. 1.3. 第三大阶段：培养阅读源码的能力
      1. 1.3.1. 第一阶段：3~4个月，（需要学会利用已有资源，达到一个加速效果，比如一些博客和书籍）
      2. 1.3.2. 第二阶段：1个月
        1. 1.3.2.1. Linux内核相关特性
        2. 1.3.2.2. 进程调度过程、线程与进程的内核实现区别、内存管理（伙伴系统与slab系统）
        3. 1.3.2.3. 进程和内存相关数据结构
        4. 1.3.2.4. 内核slab链表细节，常见的slab子项（例如task_struck，file_struct等等）
        5. 1.3.2.5. 五种IO模型，以及等待队列的意义
        6. 1.3.2.6. select、poll、epoll之间的区别，以及内核实现
    4. 1.4. 第四大阶段： 到这里，如果进展顺利的话，大概刚刚才到大三
      1. 1.4.0.1. 总结复习已有知识，查漏补缺
      2. 1.4.0.2. 完善高并发服务器，改为分布式服务器
      3. 1.4.0.3. 学会使用几个RPC组件，学会protobuf的使用等等
      4. 1.4.0.4. 学习设计模式的思维
      5. 1.4.0.5. 学习架构思维： 推荐：《大数据日知录》
      6. 1.4.0.6. 单机->多机
      7. 1.4.0.7. 分层思维
  2. 1.5. 高级数据结构知识的掌握：10~20天
    1. 1.5.0.1. 红黑树（记住特性即可）
    2. 1.5.0.2. B-tree、LSM tree、跳表、布隆过滤器等等
